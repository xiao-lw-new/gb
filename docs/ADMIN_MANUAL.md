## 3. 菜单说明（按导航分组）

> 备注：当前资源大多是只读（禁用 create/edit/delete），用于审计与查询；典型操作是“筛选 + 搜索 + 查看详情 + 复制字段（地址/交易哈希）”。

### 3.1 用户中心

- **用户管理**（`用户中心`）
  - 用途：查询用户基础信息、激活状态、上级关系
  - 关键字段：用户ID、钱包地址（可搜索/可复制）、上级ID、激活状态、注册时间
  - 筛选：仅查看激活用户

- **VIP等级管理**（`用户中心`）
  - 用途：查看用户 VIP 等级、比例、晋升类型（自动/手动锁定）
  - 关键字段：用户ID、VIP等级、收益比例、类型、更新时间

- **VIP业绩看板**（`用户中心`）
  - 用途：查看业绩指标（个人/团队/大区/小区）、激活人数（直推/团队）
  - 关键字段：个人业绩(U)、团队总业绩(U)、大区ID、大区/小区业绩、直推激活、团队激活

### 3.2 质押体系

- **质押汇总**（`质押体系`）
  - 用途：按用户、质押类型汇总持仓/价值
  - 关键字段：用户ID、类型（7天/15天）、质押U价值、代币数量、最后变动
  - 筛选：类型（7天/15天）

- **质押流水**（`质押体系`）
  - 用途：查询质押变动明细
  - 关键字段：用户ID、类型、IDX、U价值变动、代币变动、交易哈希（可复制）、区块时间

### 3.3 收益配额

- **收益额度**（`收益配额`）
  - 用途：查看用户收益配额余额与累计额度
  - 关键字段：当前剩余额度、累计总额度、更新时间

- **配额流水**（`收益配额`）
  - 用途：配额变动审计（购买/收益扣除）
  - 关键字段：类型、变动前/变动额/变动后、备注、时间
  - 筛选：类型（购买/扣除）

### 3.4 收益审计

- **静态收益记录**（`收益审计`）
  - 用途：按日查看静态收益（7天/15天）及预计/实际发放
  - 支持日期范围筛选

- **推荐奖汇总 / 推荐奖明细**（`收益审计`）
  - 汇总：按“获利用户ID + 日期”聚合展示预计/实际获得
  - 明细：按“来源用户 -> 获利用户(上级) + 代数/比例/基准金额”展示计算过程

- **VIP奖励汇总 / VIP极差明细**（`收益审计`）
  - 汇总：按“获利VIP + 日期”聚合展示预计/实际获得与发放时间
  - 极差明细：展示来源用户、获利VIP、基准收益、理论/已分配、本次极差等计算过程

### 3.5 涡轮池监控

- **转入记录(SwapIn) / 转出记录(SwapOut)**（`涡轮池监控`）
  - 用途：监控 swap 交易（入/出币种、数量、交易哈希、时间）
  - 关键字段：用户ID、交易哈希（可复制）、区块时间

## 4. 观察钱包（地址 Token 免登录，仅限授权场景）

系统支持一种“按钱包地址伪登录”的特殊 Bearer Token（**高风险能力**，务必妥善保管 suffix；建议仅用于内部排查）。

### 4.1 Token 格式

- `Authorization: Bearer _token_{address}_{suffix}`

示例：

- `Authorization: Bearer _token_0x1234...abcd_mock_token`

### 4.2 服务端开关（.env）

需要显式开启：

- `ADDRESS_TOKEN_LOGIN_ENABLED=true`
- `ADDRESS_TOKEN_LOGIN_SUFFIX=mock_token`（建议生产使用长随机串，而不是固定明文）

开启后，请求携带上述 token 将被视为 `{address}` 对应用户已登录，可直接访问原本 `auth:sanctum` 保护的接口（例如 `api/mg/rewards/flow`）。

> 生产提示：如果线上启用了 `php artisan config:cache`，修改 `.env` 后需要执行 `php artisan config:clear` 或重新 `php artisan config:cache` 才会生效。
